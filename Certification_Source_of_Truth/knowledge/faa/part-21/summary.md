# 14 CFR Part 21 Summary

## Why This Part Matters

Part 21 is the procedural authority that governs how airplanes, articles, and design changes get approved. For most certification projects, it answers the first question before any technical question: what approval path is this program using, and what does that choice trigger for the applicable basis, showing of compliance, and ICA obligations?

## Practical Role In This Repo

- Use Part 21 as the change-process backbone for TC, ATC, and STC programs.
- Use Part 21 with Part 23 for normal-category airplane programs.
- Use Part 21 with Part 25 for transport-category airplane programs.
- Treat Part 21 as the source that determines how later technical requirements get pulled forward into a project basis.

## Sections That Usually Drive Project Strategy

- Section 21.17: identifies the applicable airworthiness and noise requirements for type certification.
- Section 21.19: identifies the basic routes for special classes and other type certificate applications.
- Section 21.21: governs issuance of a type certificate.
- Section 21.31: defines technical data and the type design package.
- Section 21.50: governs instructions for continued airworthiness and other operating information.
- Section 21.93: defines when a change is major versus minor.
- Section 21.95 through 21.101: govern approvals for changed products and the applicable regulations for those changes.
- Section 21.111 through 21.120: govern the STC application and issuance path.

## Working Interpretation

For most report-building use, Part 21 should be read as the rule that frames the certification basis and approval pathway, while Part 23 or Part 25 provide the main technical standards. A project report that cites only Part 23 or Part 25 without showing its Part 21 change path is usually incomplete.
